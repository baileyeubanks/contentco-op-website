Usage Rules

Always Lowercase

The wordmark is never capitalized. "root" not "Root" or "ROOT" in the logo context.

Status Dot Required

The dot indicator is part of the mark. It communicates operational status and must always be present.

Clear Space

Maintain a minimum clear space equal to the height of the letter "o" on all sides of the wordmark.

Minimum Size

The wordmark should not appear smaller than 14px in digital or 8pt in print to maintain legibility.

Dot Color

On dark backgrounds use #4c8ef5. On light backgrounds use #4f6ee5. The dot may use status colors in functional UI contexts.

No Lockups

Do not combine the wordmark with other brand logos in a single lockup. Keep it standalone.

Guidelines

Stroke Width

Always use stroke-width: 2 for standard icons. Use 1.5 for dense UI or smaller sizes.

Color

Default icon color is var(--text-secondary). Active/selected states use var(--accent). Status icons match their respective status colors.

Sizing

Standard: 20px. Compact: 16px. Feature/empty-state: 32px or 48px. Always use even pixel values.

No Filled Icons

Root uses outlined (stroke) icons exclusively. Never use filled icon variants.

Voice & Tone

Root speaks like a senior engineer: precise, calm, and direct. No fluff, no marketing speak.

// Operational

Every message serves a function. Status updates, confirmations, and error reports are concise and actionable. The interface tells you what happened, what it means, and what to do next.

"Deploy completed. 3 services updated in us-east-1. No breaking changes detected."

// Precise

Use exact numbers, timestamps, and identifiers. Avoid vague language like "a few" or "recently." Operators need specifics to make decisions.

"Response time increased 23% over the last 4 hours (142ms avg, up from 115ms)."

// Technical but Human

Write for engineers, not robots. Use technical terms when they are the clearest option, but keep sentence structure natural. Contractions are fine. Jargon for its own sake is not.

"The auth token expired. You'll need to re-authenticate to continue."

// No-Nonsense

Skip the pleasantries in functional UI. No "Oops!" or "Uh oh!" for errors. No "Great job!" for completions. State what happened clearly and move on.

"Migration failed on step 3 of 7. See logs for details."

Do's and Don'ts

Quick reference for maintaining brand consistency across all Root touchpoints.

✓ Do

✓
Use the dark theme as the default. Root is a dark-first product.
✓
Keep border-radius at 4px. Consistency is more important than personality.
✓
Use Inter for all UI text. Use monospace for code, data values, and timestamps.
✓
Write the wordmark as "root" (lowercase) with the status dot in brand contexts.
✓
Use status colors (success, warning, error) only for their intended semantic meaning.
✓
Maintain generous whitespace. Let the interface breathe.
✓
Use the accent color (#4c8ef5) sparingly for interactive elements and focal points.

✕ Don't

✕
Don't use rounded corners beyond 4px. No pill shapes, no large radii.
✕
Don't capitalize the wordmark. "Root" or "ROOT" in logo usage is incorrect.
✕
Don't use the accent color for large background fills. It's for small focal elements only.
✕
Don't add gradients, shadows, or glows to the wordmark or icon elements.
✕
Don't use casual or playful language in the interface. No emoji in operational UI.
✕
Don't mix icon sets. Use Lucide exclusively. No mixing Font Awesome, Heroicons, etc.
✕
Don't remove the status dot from the wordmark. It's not optional.
